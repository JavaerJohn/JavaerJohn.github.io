# Java 基础

# 基础概念

# Java 语言的特点

  1. 简单性：Java 可看做是 C++ 的 "纯净" 版本，它取消了指针运算、多继承、操作符重载等难以理解的特性，且引入了垃圾的自动回收机制。

  2. 面向对象：Java 是一种面向对象的编程语言。

  3. 平台无关性：一次编写，到处运行。

  4. 支持多线程：Java 是第一个支持并发程序设计的主流语言（C++ 没有内置的多线程机制，因此必须调用操作系统的多线程功能才能进行多线程程序设计）。

  5. 安全性：Java 从一开始就被设计成了能够防范各种攻击的语言，如防止运行时堆栈溢出，防止破坏自己进程空间之外的内存，防止未经授权读写文件等。

  6. 编译与解释并存：程序运行初期，Java 虚拟机通过解释器解释执行 Javac 编译的字节码（class）文件。随着运行时间的增长，即时编译器会将热点代码编译成本地机器码，以提高系统性能。

# Java 如何实现平台无关性

  1. JVM：编译器 Javac 可以将 Java 程序编译成与计算机体系结构无关的字节码文件，字节码文件不仅可以在任何计算机上解释执行，还可以动态地转换成本地机器码，该转换过程是由 JVM 实现的，JVM 是平台相关的，它屏蔽了不同操作系统之间的差异。

  2. Java 语言规范：Java 语言保证了基本数据类型在所有平台上的一致性，如 int 类型永远是 32 位。在 C/C++ 中，基本数据类型的长度依赖于机器、编译器以及操作环境的版本等。Java 中基本数据类型有固定的的字节数，二进制数据以固定的格式进行存储和传输，字符串以标准的 Unicode 格式存储。

# JDK vs JRE vs JVM vs JIT

  1. JDK：Java 开发工具包（Java Development Kit，JDK），包含编译器 Javac、JRE 以及一些调试和分析工具（如 javadoc、jconsole 等），JDK 可创建和编译 Java 程序。

  2. JRE：Java 运行时环境（Java Runtime Environment，JRE），是运行已编译 Java 程序所必需的软件环境，包含 JVM、Java 标准类库等。JRE 面向 Java 程序的使用者而非开发者，它不能创建 Java 程序。

  3. JVM：Java 虚拟机（Java Virtual Machine，JVM），用来运行 Java 字节码文件，JVM 是 Java 跨平台的关键。

  4. JIT：即时编译器（Just In Time Compiler，JIT），存在于 JVM 中，负责将热点代码编译成本地机器码，提高程序运行的效率。

# 自动装箱与自动拆箱

Java 中每个基本数据类型都对应一个包装类，自动装箱指的是将基本数据类型包装为一个包装类对象，例如向一个泛型类型为 Integer 的集合添加 int 元素。自动拆箱指的是将一个包装类对象转换为一个基本数据类型，例如将一个 Integer 对象赋值给一个 int 类型的变量。

# 泛型

泛型本质上是参数化类型，主要用来解决不确定对象具体类型的问题。泛型提供了编译时的类型安全检测机制，该机制允许程序员在编译时检测到非法的类型。泛型的好处包括：

  1. 参数安全，放进去什么类型，取出来就是什么类型。

  2. 可读性高，编码阶段就可以显式知道泛型集合、泛型方法等处理的对象类型。

  3. 代码重用，可以合并同类型数据的处理代码。

# 泛型擦除

Java 中的泛型主要用于编译阶段，编译后的字节码文件不包含泛型类型信息，例如 List<Integer> 和 List<String> 在编译后都会变成 List（泛型参数只用于编译阶段的类型检查，不作用于运行阶段）。

# 按值调用与按引用调用

  1. 按值调用指的是方法接收调用者提供的值。

  2. 按引用调用指的是方法接收调用者提供的变量地址。

Java 是按值调用的，当向方法传递基本数据类型时，方法接收到的其实是参数值的副本。传递对象时，方法接收到的是对象引用的副本，对于引用类型而言，生成的副本同样指向原有对象。

# String 对象是否可以修改

不可以修改，因为 String 类和其存储数据的 char/byte 数组都是被 final 修饰的（JDK9 中使用 byte 数组）。对一个 String 对象的任何修改实际上都是创建一个新 String 对象，再引用该对象。

JDK 1.8 及以前 String 使用 char 数组存储数据，JDK 1.9 及以后使用 byte 数组。

# 字符串拼接的方式

  1. 使用 + 拼接符。

  2. 使用 StringBuilder 或 StringBuffer，调用 append 方法。

+ 拼接符的底层是先 new 一个 StringBuilder 对象，然后调用 append 方法，之后再执行 toString 方法。+ 与 StringBuilder 的执行效率可能会存在差异，如果是拼接多个字符串常量，那么 + 的效率会更高一些，因为编译器会进行优化，例如将 "a" + "b" 直接视为 "ab"。如果拼接不同的变量，则需要视情况而定，若不同变量写在同一行，那么 + 与 StringBuilder 几乎相同。若写在不同行，那么 + 会额外创建新的 StringBuilder 对象，例如在 for 循环的循环体中使用 +。

# 浅拷贝与深拷贝

  1. 浅拷贝仅拷贝原对象中的基本数据类型和引用变量，不拷贝引用变量指向的实际对象，因此修改拷贝对象可能会影响到原对象（不安全）。

  2. 深拷贝指的是完全拷贝基本数据类型和引用类型，拷贝对象与原对象完全独立（安全）。

# 异常的分类

Java 中所有异常都是 Throwable 的子类，异常可分为 Error 和 Exception：

  1. Error：Error 是 Java 运行时系统的内部错误或资源耗尽错误，如 StackOverflowError 和 OutOfmemoryError，这种异常程序无法处理。

  2. Exception 可分为受检异常和非受检异常：

    1. 受检异常必须在代码中显式处理，否则编译时会出错，如 IOException、FileNotFoundException 等。
    2. 非受检异常是运行时异常，继承自 RuntimeException，如 IndexOutOfBoundsException、NullPointerException 等。

# Class 类的作用

Class 类是 Java 反射机制的基础，程序运行时，虚拟机会将 .java 文件编译成的 .class 字节码文件加载到内存，并生成对应的 Class 对象。通过 Class 对象的方法，我们可以获取到类的相关信息，如类名、属性、方法、构造方法、父类、接口以及注解等。

获取 Class 对象的三种方法：① 对象.getClass()；② 类名.class；③ Class.forName(类的全限定名)。

# JDK 1.8 的新特性

  1. Lambda 表达式：允许把函数作为参数传递给方法，可简化匿名内部类的代码。

  2. 函数式接口：使用 @FunctionalInterface 标识，有且仅有一个抽象方法（可包含默认方法、静态方法以及私有方法），使用时可转化为 lambda 表达式（@FunctionalInterface 可以检测接口是否为函数是接口）。

  3. 方法引用：可以直接引用已有类或对象的方法或构造方法，进一步简化 lambda 表达式（方法引用可看作 Lambda 表达式的另一种表现形式）。方法应用的调用示例有：

    1. 引用静态方法：类名::静态方法名，如 String::valueOf。
    2. 引用实例方法：对象名::方法名，如 user::getName。
    3. 应用构造方法：类名::new，如 User::new。

  4. 默认/静态方法：接口可以定义 default 修饰的默认方法，降低接口升级的复杂性，还可以定义静态方法。

  5. 注解：引入重复注解机制，相同注解可以在同一个地方声明多次。注解的范围也进行了扩展，可作用于局部变量、泛型、方法、异常等。

  6. Stream：Stream API 可对包含一个或多个元素的集合做各种操作，包括中间操作 和 终端操作（终端操作 会返回一个结果，而 中间操作 会返回一个 Stream 流）。

  7. Optional 类：处理空指针异常。

  8. Date API：加强对日期与时间的处理。

# 面向对象

# 如何理解面向对象编程设计

结合面向过程来谈一谈我对面向对象的理解，面向过程是让计算机按照步骤有序的做一件事，它会将大的程序拆分成多个小的过程/函数，然后通过调用各个子过程/子函数来实现相应的功能。面向对象通常会将现实中的事物进行抽象并映射为开发对象，对象内部定义了该对象的属性（成员变量）和行为（方法），主程序可通过调用对象内的方法来解决目标问题。比如开门这个动作，面向过程会将 door 对象作为参数传递给 open(Door door) 方法，而面向对象则会在 door 对象内部定义 open 行为，然后直接调用 door.open() 即可。

面向对象相比于面向过程的优势在于：易维护、易复用、易扩展。面向对象更适合处理一些比较复杂的业务逻辑，因为它可以降低程序的耦合度，提高系统的灵活性。面向过程相比于面向对象也有一定的优势，例如它的开销更低，性能更高。

函数和过程的区别主要在于有无返回值，函数/过程是最小的模块封装单位。参考自什么是面向过程编程，面向对象编程open in new window。

# 面向对象的三大特征

  1. 封装指的是将对象的属性隐藏在对象内部，不允许外部对象直接访问，也就是对对象的使用者隐藏了数据的实现方式，但使用者可通过对象提供的方法来访问这些属性。

  2. 继承：继承是使用已有的类快速构建新类的一种技术，子类不仅拥有父类的属性和行为（包括 private 成员，private 成员会被隐藏且不可访问），还可以添加一些新的功能。通过继承，我们可以复用以往的代码，提高开发的效率。

  3. 多态：引用变量所指向的具体类型以及该引用变量调用的方法在编译时并不确定，而是在程序运行期间才能确定，具体表现为父类的引用指向子类的实例。

多态的注意事项：

  • 声明类型和实际类型之间具有继承（类）/实现（接口）的关系。

  • 引用变量调用的到底是哪个类中的方法，必须在程序运行期间才能确定。

  • 多态不能调用只在子类存在但在父类不存在的方法。

  • 如果子类重写了父类的方法，真正执行的是子类覆盖的方法，如果子类没有覆盖父类的方法，执行的是父类的方法。

# 重载和重写

  1. 重载指的是方法名称相同，但参数类型或参数个数不同。对编译器来说，方法名和参数列表共同组成了一个唯一键，称为方法签名，JVM 通过方法签名来决定调用哪种重载的方法。由于重载在编译时就可以根据规则确定调用哪个目标方法，因此重载属于静态绑定。JVM 在重载方法中选择合适方法的顺序为：
    1. 精确匹配。
    2. 基本数据类型自动转换成更大表示范围。
    3. 自动拆箱与装箱。
    4. 子类向上转型。
    5. 可变参数。

注意，方法的返回值不能作为判断方法之间是否构成重载的依据。

  1. 重写指的是子类在实现接口或继承父类时，保持方法签名完全相同，但实现了不同方法体。子类在重写父类的方法时需要注意：
    1. 方法的返回值类型、方法名、参数列表必须与父类相同，抛出的异常范围要小于等于父类，访问修饰符范围要大于等于父类。
    2. 如果父类方法访问修饰符为 private/final/static，那么子类就不能重写该方法，但是被 static 修饰的方法能够被再次声明。
    3. 构造方法无法被重写。

# Object 类有哪些方法

  1. equals：判断两个对象是否相等，默认使用 == 进行比较，可以重写该方法自定义比较规则。

equals 方法规范：自反性、对称性、传递性、一致性、对于任何非空引用 x，x.equals(null) 均返回 false。

  1. hashCode：返回一个整数类型的散列码，其值与对象的内存地址有关。

  2. toString：打印对象的相关信息。

  3. clone：用于克隆对象。

  4. finalize：在对象被回收之前调用，可用来释放一些资源。

  5. getClass：返回包含对象信息的类对象。

  6. wait/notify/notifyAll：阻塞或唤醒线程。

# 内部类

内部类的作用：

  1. 内部类可以访问定义这个内部类的作用域中的数据。

  2. 内部类可以对同一包中的其他类隐藏（private）。

  3. 内部类可以解决 Java 单继承的缺陷。

当前类中可定义多个内部类，每个内部类都可继承不同的外部类，这样当前类就近似拥有了多个外部类的属性和行为。

内部类分为：

  1. 静态内部类（类比静态成员变量）：

由 static 修饰，作用域仅限在包内，可以通过 外部类名.内部类名 的方式来声明（在当前包下的其它类中可以通过此种方式创建静态内部类，在包外则不行），类内只能访问外部类的静态成员变量和方法。

静态内部类若要访问外部类的实例成员变量和方法，可以先创建外部类的实例，然后再进行调用。另外，外部类也无法直接访问静态内部类的实例成员变量和方法，类似的，可以先创建内部类的实例，然后再访问。

// Outer 类为外部类
public class Outer {
    // 静态内部类
    static class Inner {
    }
}

// OtherClass 类与 Outer 类在同一包下
public class OtherClass {
    public static void main(String[] args) {
        // 创建 Inner 对象
        Outer.Inner inner = new Outer.Inner();
    }
}
  1. 成员内部类（类比实例成员变量）：

成员内部类的创建依赖于外部类对象，在外部类的实例创建之前是无法创建成员内部类对象的。成员内部类的作用域也仅限在包内，类内只能定义实例成员变量和实例方法，但其可以访问外部类中的所有成员。

// Outer 类为外部类
public class Outer {
    // 成员内部类
    class Inner {
    }
}

// OtherClass 类与 Outer 类在同一包下
public class OtherClass {
    public static void main(String[] args) {
        // 创建 Inner 对象
        Outer outer = new Outer();
        Outer.Inner inner = outer.new Inner();
    }
}
  1. 局部内部类（类比局部变量）：

局部内部类定义在方法内，不能声明访问修饰符（public、protected、private 以及 static），只能定义实例成员变量和实例方法，且作用域仅限于声明类的代码块中，局部内部类可访问外部类的所有成员。

public class Outer {
    public void test() {
        // 局部内部类
        class Inner {
        }
        // 创建 Inner 对象
        Inner inner = new Inner();
    }
}
  1. 匿名内部类：

只使用一次的、没有名字的内部类，可以使用 Lambda 表达式简化匿名内部类的实现（匿名内部类必须继承一个抽象类或实现一个接口）。

public class Outer {
    public void test() {
        // 匿名内部类, 该类实现了 OneInterface 接口
        OneInterface inner = new OneInterface() {
            @Override
            public void test() {
                System.out.println("Hello");
            }
        };
        // 等效于
        // OneInterface inner = () -> System.out.println("Hello");
    }
}

interface OneInterface {
    void test();
}

    }
}

查看外部类编译后的 .class 文件可以发现，匿名/局部内部类会将外部类对象以及所要访问的局部变量赋值给自己的成员变量（这也就能解释为什么匿名/局部内部类可以访问外部类的所有成员了，因为其成员变量就指向外部类对象）。若局部变量为基本数据类型，那么内部类的成员变量也为基本数据类型，且二者的值相同。若局部变量为引用类型，那么该局部变量与内部类的成员变量将指向同一个对象。如果局部变量不被 final 修饰，那么当局部变量的值改变或者局部变量指向了另一个对象时，内部类的成员变量并不会做出相应的修改，也就是说，该成员变量的值仍保持不变或仍指向原先的对象，这将导致局部变量和内部类中的数据不同步。加上 final 后，局部变量和内部类的成员变量始终保持相同的值或始终指向同一个对象，从而解决了数据不同步的问题。

参考自 Java 局部内部类访问局部变量为什么必须加 final 关键字open in new window、匿名内部类访问局部变量为什么要加 final 修饰open in new window。

# 接口和抽象类的区别

  1. 成员变量：抽象类无特殊需求；接口的成员变量默认被 public static final 修饰。

  2. 构造方法：抽象类有构造方法，不能实例化；接口没有构造方法，不能实例化。

  3. 方法：抽象类可以没有抽象方法，但有抽象方法一定是抽象类；接口的方法默认被 public abstract 修饰，JDK 1.8 中支持默认方法和静态方法，JDK 1.9 中支持私有方法。

  4. 继承：抽象类只能单继承；接口可以多继承。

# 类初始化的顺序

父类的静态代码块和静态变量 → 子类的静态代码块和静态变量 → 父类的普通代码块和普通变量 → 父类的构造方法 → 子类的普通代码块和普通变量 → 子类的构造方法。
