# Java 集合

# List

# ArrayList 简介

ArrayList 是基于数组来实现的，它的数据域是一个名为 elementData 的数组，elementData 的容量大于等于 ArrayList 实际存储元素的个数 size，当 size 达到 elementData 的容量时，就会触发 ArrayList 的扩容机制，也就是创建更大的数组（新数组容量是原数组的 1.5 倍）并将原数组中的元素复制到新数组中（调用 Arrays.copyOf 方法进行复制）。此外，elementData 被 transient 修饰，序列化时仅序列化真正有实际值的数组元素。

ArrayList 的优点是支持对元素的快速随机访问，可以根据索引快速获取到对应的元素，ArrayList 实现了 RandomAccess 接口，该接口是一个标志接口，可用于区分 ArrayList 和 LinkedList，如果一个集合类实现了 RandomAccess，那么该集合通过索引遍历比迭代器更快。

Collections 类的 binarySearch 方法根据 list instanceof RandomAccess 的结果来决定采用索引获取元素还是采用迭代器。

ArrayList 的缺点是在列表首部或中间插入或删除元素的速度会比较慢，因为需要频繁移动大量元素的位置。

# ArrayList 和 Arrays.asList

Arrays 类的 asList 方法返回的是一个 ArrayList 对象，但是这个 ArrayList 并不是 java.util.ArrayList，而是 Arrays 类的静态内部类，该内部类继承了 AbstractList 类并实现了 RandomAccess 接口，但它不能直接调用 add 方法和 remove 方法，因为其并未重写 AbstractList 的 add 和 remove 方法。

可以执行 ArrayList(Arrays.asList(array)) 来生成 java.util.ArrayList 对象，其中 array 为一个数组。

# LinkedList 简介

LinkedList 是基于双向链表来实现的，链表中的每个节点都会记录它的前置节点和后置节点。相比于 ArrayList，LinkedList 在列表首部插入或删除元素的速度会更快，因为只需要修改几个指针的值。但是如果在列表中间（既不靠近队首也不靠近队尾）插入或删除元素，那么当元素数量非常大时，LinkedList 的执行效率会非常低，甚至可能远低于 ArrayList。此外，LinkedList 也不支持对元素的快速随机访问，每次访问元素时，都需要遍历一次链表。

LinkedList 实现了 Deque 接口，这使得 LinkedList 具备了队列和栈的性质。由于 LinkedList 不需要连续的内存空间进行存储，因此其内存利用率比较高。

如果插入或删除的位置在列表中间，那么当元素的数量较多时，LinkedList 性能的消耗主要来自于遍历链表所需要的时间开销，ArrayList 则来自于移动元素的开销，先比之下，ArrayList 的效率更高。如果位置在列表尾部，那么二者的效率差不多，因为 ArrayList 需要移动的元素较少，而 LinkedList 可以从尾部开始向前遍历。如果位置在列表首部，则 LinkedList 的效率更高。

引申问题：ArrayList 和 LinkedList 谁更占用内存？

答案是不确定，因为 LinkedList 的 Node 节点相比 ArrayList 中的单个元素更占空间，但 ArrayList 存在扩容机制，可能刚刚完成扩容就不再添加新元素了，那么多出的一半数组就会被浪费掉。

# fail-fast 机制

fail-fast 机制是所有集合类（实现 Collection 接口的类）都有的一种错误机制，这些类通过引入变量 modCount 来记录集合结构性变化的次数，所有涉及结构变化的方法都会增加 modCount 的值。当迭代器初始化时，modCount 会被赋值给 expectedModCount，以后每次通过迭代器访问元素时都会检查最新的 modCount 与 expectedModCount 是否相等，不相等的话就会抛出异常。因此当多个线程同时操作同一个集合中的元素时，就可能产生 fail-fast 事件。

# HashMap 简介

HashMap 是用来存储键值对的数据结构，JDK 1.7 的底层实现是数组 + 链表，JDK 1.8 改为数组 + 链表/红黑树，节点类型也由原先的 Entry 节点改为了 Node 节点，但无论是 Entry 还是 Node，它们都包含四个成员变量：key、value、next 指针和 hash 值。

HashMap 主要成员变量包括存储数据的 table 数组、元素数量 size、加载因子 loadFactor。table 数组的每个下标对应一条链表/红黑树，存储元素时，根据元素的 key 计算哈希值，并根据哈希值计算数组的下标，所有哈希值冲突的元素都会被存放到同一个哈希槽中，JDK 1.7 中采用头插法的方式存储元素，JDK 1.8 改为了尾插法，并且同一个哈希槽中的元素达到一定阈值后会将链表转化为红黑树。哈希冲突会影响 HashMap 查询的效率，所以 key 的 hash 值要尽可能分散。

HashMap 默认初始化容量为 16，扩容容量必须是 2 的幂次方、最大容量为 1<< 30 、默认加载因子为 0.75。
