MyBatis 实现分页查询的三种方式

前言

分页查询是许多网站都具备的一个基本功能，其作用是将查询到的结果集做分页展示，例如每页显示 100 条数据。当数据量较大时，分页查询可以减轻服务端的查询压力以及前端的渲染压力。MyBatis 提供了三种分页查询的方式，下面我们依次介绍其实现过程。

首先创建实体类 User：

@Data
public class User {

    private int id;

    private String userName;

    private int age;

    private String address;

    private Date createTime;

    private Date updateTime;
}

然后创建 user 表：

DROP TABLE IF EXISTS `user`;
CREATE TABLE `user`  (
  `id` int(11) NOT NULL AUTO_INCREMENT,
  `user_name` varchar(255) CHARACTER SET utf8mb4 COLLATE utf8mb4_0900_ai_ci NOT NULL,
  `age` int(11) DEFAULT NULL,
  `address` varchar(255) CHARACTER SET utf8mb4 COLLATE utf8mb4_0900_ai_ci DEFAULT NULL,
  `gmt_create` datetime(0) DEFAULT NULL,
  `gmt_modified` datetime(0) DEFAULT NULL,
  PRIMARY KEY (`id`) USING BTREE
) ENGINE = InnoDB AUTO_INCREMENT = 16 CHARACTER SET = utf8mb4 COLLATE = utf8mb4_0900_ai_ci ROW_FORMAT = Dynamic;

Limit 分页

Limit 是 MySQL 提供的分页查询功能，其语法为：

limit ${offset}, ${rows}
# 或
limit ${rows} offset ${offset}

参数 offset 和 rows 分别指定检索的偏移量和记录数，例如 limit 1, 5 表示查询从第二行到第六行的 5 条记录。

创建 Page 类：

//@Data
public class Page<T> implements Serializable {

    private final Integer RANGE = 2;

    private final Integer DEFAULT_PAGE_INDEX = 1;

    private final Integer DEFAULT_PAGE_SIZE = 3;

    // 总的记录数
    private Integer count;

    // 当前页码, 默认为 1
    private Integer pageIndex = DEFAULT_PAGE_INDEX;

    // 每页显示的记录数, 默认为 5
    private Integer pageSize = DEFAULT_PAGE_SIZE;

    // 总的页数
    private Integer total;

    // 数据内容
    private List<T> data;

    // 根据页码计算偏移量
    public Integer getStart() {
        return (pageIndex - 1) * pageSize;
    }

    // 计算总共的页数
    public Integer getTotal() {
        int total = count / pageSize;
        if (count % pageSize != 0) {
            total++;
        }
        return total;
    }

    // 前端展示的页码范围
    public Integer getFrom() {
        int from = pageIndex - RANGE;
        return Math.max(from, 1);
    }

    public Integer getTo() {
        int to = pageIndex + RANGE;
        int total = getTotal();
        return Math.min(to, total);
    }
}

Mapper 接口：

List<User> findUserPage(@Param("condition") Map<String, Object> condition, @Param("start") Integer start, @Param("pageSize") Integer pageSize);

mapper 文件：

<resultMap id="UserMap" type="User">
    <id column="id" jdbcType="INTEGER" property="id"/>
    <result column="user_name" jdbcType="VARCHAR" property="userName"/>
    <result column="age" jdbcType="INTEGER" property="age"/>
    <result column="address" jdbcType="VARCHAR" property="address"/>
    <result column="gmt_create" jdbcType="DATE" property="createTime"/>
    <result column="gmt_modified" jdbcType="DATE" property="updateTime"/>
</resultMap>

<select id="findUserPage" parameterType="Map" resultMap="UserMap">
    select
    *
    from user
    <where>
        <if test="condition.max_age != null and condition.max_age != ''">
            <![CDATA[age <= #{condition.max_age}]]>
        </if>
    </where>
    limit #{start}, #{pageSize}
</select>

<select id="findUserCount" parameterType="Map" resultType="Integer">
    select
    count(1)
    from user
    <where>
        <if test="condition.max_age != null and condition.max_age != ''">
            <![CDATA[age <= #{condition.max_age}]]>
        </if>
    </where>
</select>

使用 Limit 实现分页查询时，我们需要额外添加一个返回总记录数的 SQL，以便计算总的页码数。

测试：

@Test
public void findUser() {
    Map<String, Object> map = new HashMap<>();
    map.put("max_age", 100);
    Page<User> page = new Page<>();
    page.setPageIndex(2);
    page.setData(userMapper.findUserPage(map, page.getStart(), page.getPageSize()));
    page.setCount(userMapper.findUserCount(map));
    System.out.println(page);
}

RowBounds 分页

RowBounds 是 MyBatis 提供的一个分页类，它可以帮助我们省略 limit 子句的书写，使我们只需要关注业务层的分页逻辑即可。

Mapper 接口：

List<User> findUserByRowBounds(@Param("condition") Map<String, Object> condition, @Param("RowBounds") RowBounds rowBounds);

mapper 文件：

<select id="findUserByRowBounds" parameterType="Map" resultMap="UserMap">
    select
    *
    from user
    <where>
        <if test="condition.max_age != null and condition.max_age != ''">
            <![CDATA[age <= #{condition.max_age}]]>
        </if>
    </where>
</select>

测试：

@Test
public void findUser() {
    Map<String, Object> map = new HashMap<>();
    map.put("max_age", 100);
    RowBounds rowBounds = new RowBounds(1, 5);
    System.out.println(userMapper.findUserByRowBounds(map, rowBounds));
}

RowRounds 类的构造方法接收两个参数，分别为 offset 和 limit：

public RowBounds(int offset, int limit) {
    this.offset = offset;
    this.limit = limit;
}

offset 表示查询的偏移量，limit 表示查询的记录数，这两个参数与 Limit 查询中的两个参数的含义相同。

RowRounds 一般不推荐使用，因为它属于逻辑分页查询，即一次性从数据库中查询出所有的数据并将其存储在服务器内存，然后从结果集中检索出分页的数据。此种查询方式的执行效率较低，内存占用率较高。

物理分页指的是从数据库中查询出指定偏移量后的指定条数据。

分页插件 PageHelper

PageHelper 是一个非常实用的分页插件，其实现原理是通过拦截器将 limit 信息拼接到 SQL 语句中，从而实现分页的功能。此外，PageHelper 也免去了 count 语句（查询总记录数）的书写。

pom.xml 文件中添加依赖：

<dependency>
    <groupId>com.github.pagehelper</groupId>
    <artifactId>pagehelper</artifactId>
    <version>5.1.7</version>
</dependency>

配置 MyBatis 的核心配置文件：

<plugins>
    <plugin interceptor="com.github.pagehelper.PageInterceptor" />
</plugins>

Mapper 接口：

List<User> findUserByPageHelper(@Param("condition") Map<String, Object> condition);

mapper 文件：

<select id="findUserByPageHelper" parameterType="Map" resultMap="UserMap">
    select
    *
    from user
    <where>
        <if test="condition.max_age != null and condition.max_age != ''">
            <![CDATA[age <= #{condition.max_age}]]>
        </if>
    </where>
</select>

测试：

@Test
public void findUser() {
    Map<String, Object> map = new HashMap<>();
    map.put("max_age", 100);
    PageHelper.startPage(2, 5);
    System.out.println(userMapper.findUserByPageHelper(map));
}

PageHelper 的 startPage 方法接收两个参数，分别为 pageNum 和 pageSize：

public static <E> Page<E> startPage(int pageNum, int pageSize) {
    return startPage(pageNum, pageSize, DEFAULT_COUNT);
}

pageNum 表示页码，pageSize 表示每页的记录数。

PageHelper 也可以自定义 count 语句。
