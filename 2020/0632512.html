基本介绍

Git 是目前最流行的版本控制工具，熟练使用 Git 是每个程序员必备的技能。奈何自己非科班出身，对 Git 的使用少之又少，因此撰写本篇博文来记录常用的 Git 指令，以作备忘。

查看分支

# 查看当前分支
git branch

# 查看远程分支
git branch -r

# 查看所有分支
git branch -a

新建分支

# 1. 新建本地分支，但不会切换到该分支，如果在 xxx 分支下执行该命令，那么 new_branch 分支的代码与 xxx 分支相同
git branch new_branch

# 2. 切换分支
git checkout new_branch

# 3. 上述两条命令等效于如下命令（新建本地分支，并切换到该分支）
git checkout -b new_branch

# 4. 基于 master 分支的代码创建新分支 new_branch，并切换到该分支
git checkout master
git checkout -b new_branch

# 5. 基于远程分支 remote_branch 的代码创建新分支 new_branch，并切换到该分支
# 法一：先创建再拉取
git checkout -b new_branch
git push origin remote_branch
# 法二：上述两条命令等效于如下命令
git checkout -b new_branch origin/remote_branch

# 6. 远程仓库创建了新的分支，但本地仓库并未同步到（使用 git branch -a 获取不到新创建的分支），可先执行如下命令，然后就能看到新的分支了
git fetch

查看关联信息

# 查看本地分支与远程分支的对应关系（使用下面三条命令中的任何一个）
git branch -vv
git remote show origin
cat .git/config

提交代码

# 1. 提交代码到本地缓冲区
# 1.1 逐个添加被修改的文件
git add file1.java file2.java file3.xml
# 1.2 添加所有被修改的文件
git add .

# 2. 提交代码并添加修改说明
git commit -m '修改说明'

# 3. 确保当前分支 current_branch 上的代码是最新的，即拉取最新的代码（可省略该步骤，若不是最新的代码，git 会返回提示信息）
git pull

# 4. 将代码推送到远程分支
git push origin current_branch

合并分支

# 将当前分支 current_branch 合并到 master
git checkout master
git merge current_branch

删除分支

# 1. 删除本地分支 current_branch（不能处于即将被删除的分支，需要切换到其它分支）
git branch -d current_branch

# 2. 删除远程分支 remote_branch
git push origin -d remote_branch

修改分支名称

# 1. 将本地分支 old_branch 和远程分支 old_branch 的名称均修改为 new_branch
# 1.1 切换到 old_branch 分支下
git checkout old_branch
# 1.2 确保本地分支的代码是最新的
git pull origin old_branch
# 1.3 修改本地分支的名称
git branch -m old_branch new_branch
# 1.4 删除远程分支
git push origin --d old_branch
# 1.5 将本地分支推送到远程仓库
git push origin new_branch
